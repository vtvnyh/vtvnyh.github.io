义务论：不可逾越的代码红线

Deontology: Rules Over Results

与功利主义不同，义务论（Deontology）不关心结果的“好坏”，只关心行为本身的“对错”。在康德哲学中，某些行为（如杀害无辜）在任何情况下都是错误的，无论它能拯救多少人。

核心定义：
道德源于对规则和义务的遵守。在自动驾驶中，这意味着车辆必须遵守绝对的禁令（Hard Constraints），例如“严禁主动撞击行人”，即使这样做能避免更大的灾难。

01. 算法实现：硬约束与优先级

在代码层面，义务论不像功利主义那样进行加权求和，而是表现为优先级队列或硬性拦截器。

// 义务论逻辑伪代码 if (action == ACTIVE_HARM_TO_HUMAN) { return FORBIDDEN; // 绝对禁止，无论后果如何 } else if (action == VIOLATE_TRAFFIC_RULE) { return LOW_PRIORITY; } else { optimize_comfort(); }

这种逻辑下，车辆不会计算“撞1个人比撞5个人好”，而是首先排除所有“主动撞人”的选项。如果所有选项都包含撞人，系统可能会陷入逻辑死锁，或者默认选择“不作为”（即保持当前轨迹，哪怕结果是悲剧）。

02. “不作为”的道德责任

义务论在自动驾驶中面临的最大指责是：不作为是否也是一种恶？

  • 电车难题变体： 如果车子刹车失灵，前方有5人，旁边有1人。功利主义会转向撞1人；义务论可能认为“转向”是主动杀人，而“直行”是自然发生的悲剧，因此选择直行。
  • 法律辩护优势： 从法律角度看，义务论策略更容易为车企辩护——“车辆严格遵守了交通法规和不得伤人的底线，事故是由外部不可抗力导致的”。

03. 现实困境：规则的冲突

现实世界充满了规则冲突，简单的义务论代码难以处理复杂场景：

经典冲突：
为了避让突然冲出的儿童（规则A：保护生命），车辆不得不压实线逆行（规则B：遵守交规）。在义务论框架下，哪条规则具有更高的“绝对性”？这需要预先设定复杂的规则层级（Rule Hierarchy）。

如果规则层级设定不当，车辆可能在极端情况下表现出僵化甚至荒谬的行为，例如为了不被追尾而拒绝紧急避让行人。

04. 结语

义务论为AI设定了道德底线，防止其沦为冷血的计算器。但它缺乏灵活性，难以在“两难”中做出最优解。未来的伦理算法很可能是混合式的：以义务论为边界，在安全空间内使用功利主义进行优化。
